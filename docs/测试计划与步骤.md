# AcademicReviewer 测试计划与步骤

> **目标**：让团队同事能够按照明确步骤逐一验证系统各项功能，确保上线前所有模块稳定可用。
>
> **预计耗时**：单人完整测试约 **2~3 小时**（含等待 LLM 响应）；多人并行约 **1 小时**。

---

## 一、测试前准备（所有人必做）

### 1.1 环境检查清单

| # | 检查项 | 命令 | 通过标准 |
|---|--------|------|----------|
| 1 | Python 版本 | `python --version` | 3.10 或更高 |
| 2 | 依赖安装 | `pip install -r requirements.txt` | 无报错 |
| 3 | `.env` 配置 | 确认文件存在且 API Key 非占位值 | 至少 1 个 Key 有效 |
| 4 | 数据目录 | 启动后自动创建 `data/` | 系统自动处理 |

### 1.2 启动服务

在两个终端分别执行：

**终端 1 — FastAPI 后端：**
```bash
python run.py
```
验证：浏览器打开 `http://127.0.0.1:8000/docs`，应看到 Swagger UI 文档页面。

**终端 2 — Gradio 前端：**
```bash
python app/gradio_app.py
```
验证：浏览器打开 `http://127.0.0.1:7860`，应看到 4 个标签页界面。

### 1.3 测试文件准备

准备以下测试材料（`tests/fixtures/sample_isef_paper.txt` 可复用）：

| 编号 | 用途 | 建议 |
|------|------|------|
| 文件A | ISEF 科研论文 (.txt) | 使用已提供的 sample_isef_paper.txt |
| 文件B | John Locke 议论文 (.txt) | 自己写一段 800 字的观点论述 |
| 文件C | 有引用问题的论文 (.txt) | 正文引用 [1][2] 但参考文献缺一条 |
| 文件D | 同一论文 (.pdf 或 .docx) | 测试格式兼容性 |
| 文件E-G | 3 篇获奖 + 3 篇失败文章 | 用于校准引擎测试 |
| 文件H-I | 2 篇教师经验文档 | 使用 data/expert_insights/ 下的示例文档 |

## 五、给测试同事的快速上手建议

1. **首次测试建议顺序**：测试组 1 → 测试组 2 → 测试组 9 → 其他各组
2. **多人分工建议**：
   - 同事A：负责组 2（核心评审）+ 组 3（多竞赛）
   - 同事B：负责组 4（文件格式）+ 组 5（评审历史）+ 组 8（异常边界）
   - 同事C：负责组 6（模型切换）+ 组 7（校准引擎）+ 组 7.5（教师经验解析）
3. **注意**：组 2 和组 3 的测试会消耗 API 费用，建议先用最便宜的模型（如 DeepSeek）跑通，再测试其他模型
4. **校准引擎**大部分测试不需要 API Key（组 7 零成本自由测试，组 7.5 的 Mode 3 零成本，Mode 1/2 需要 API Key）
5. 如果某项测试失败，请优先排查 `.env` 配置和网络连接
