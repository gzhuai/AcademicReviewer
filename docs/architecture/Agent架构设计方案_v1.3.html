一、核心定位与设计原则

核心定位：「AI 学术导师 + 质量门禁」。在不替代学生思考的前提下，系统性地发现文稿中的问题，并以可操作的反馈指导学生自主改进。

六项设计铁律

原则说明
学生主权AI 只做批注和建议，绝不直接改写正文
分层审查从宏观（论点结构）到微观（语法拼写），逐层把关
竞赛对齐每次审查与具体竞赛的评分标准严格对标
可解释性每个建议必须附带理由和评分维度
迭代闭环支持多轮提交→审查→修改→再审
数据驱动校准用获奖/失败样本持续反推评委隐性偏好（v1.3 新增）

2.3 校准管线：Calibration Engine

纯统计管线，不调用 LLM。季后批量离线运行，从获奖/失败文章的差异中提取量化校准信号。

核心方法论：Cohen's d 效应量

d = (M_winners - M_losers) / SD_pooled

d > 0.8  → 大效应 — 决定性差异 → critical 项
d > 0.5  → 中效应 — 重要差异 → major 项
d > 0.2  → 小效应 — 轻微差异 → minor 项
d < 0.2  → 无差异 — 评委可能不关心 → 不写入

四阶段处理流程

Step 1: 特征提取 — 21 维文本特征扫描 → 特征矩阵（篇×特征）
Step 2: 分组差异分析 — Cohen's d + 交叉验证（W vs L, W vs Ext）
Step 3: 规则生成 — d>0.8→critical, d>0.5→major, 失败独有→fatal_defects
Step 4: Diff 生成 — 与上赛季配置对比，人工友好变更建议

2.4 各 Agent 校准需求矩阵

Agent需校准？深度方式
A1 Rubric✅深维度权重反推、子维度发现
A2 Structure✅中浅二元检查升级为「有+质量预期」
A3 Argument✅✅ 强需最深论证模式反推、fatal_defects 挖掘
A4 Style✅中风格特征区间统计
A5 Citation❌—规则驱动，无隐性偏好
A6-A7❌—L0 通用引擎

解读：A3 是校准的重中之重。A1 校准的是「评分标准」（评委声称看重什么），A3 校准的是「评分实质」（评委实际看重什么），两者缺一不可。

三、工作流设计

3.1 审查工作流（在线）

Stage 1: 材料摄入 → Stage 2: Multi-Agent 审查（3轮并行）→ Stage 3: 反馈报告 → Stage 4: 学生修改辅助 → Stage 5: 终审导出

3.2-3.3 赛季校准工作流（离线）

时期人的操作系统动作
赛季前（赛前 2-3 月）上传官方 rubric → 标记样本集 → 触发校准 → 审核建议 → 锁定配置特征提取、Cohen's d、交叉验证、校准建议、版本快照
赛季中零操作按锁定配置运行审查管线
赛季后（结果公布后）标注获奖/失败名单 → 审核新校准建议增量校准、生成新草案

质量指标：系统预测分数与实际获奖排名的 Spearman ρ。ρ > 0.8 → 锁定；ρ 0.6-0.8 → 微调；ρ < 0.6 → 大修。

四、技术实现建议

  • 结构化 Prompt 模板：每 Agent 至少含 [Role] / [Context] / [Task] / [Output Format] / [Constraints]
  • 模型层抽象：同时支持 OpenAI / Claude / DeepSeek / Qwen 等，通过适配层切换
  • 长文本处理：分段处理 + 跨段上下文窗口

模块功能优先级

五、风险与边界

风险应对策略
学生过度依赖 AI只做批注不改写；导师最终确认；完整审查日志
AI 产生幻觉引用引用建议须经 API 校验；虚假引用标为 Critical
不同竞赛要求冲突每个竞赛维护独立 rubric 模板
学生 voice 丧失A6 专门守护原创性；风格建议标注为可选
小样本校准偏差n<10 降低 confidence；n<5 不做校准
评委偏好年度漂移追踪 Spearman ρ 趋势；连续两季 <0.6 触发重校准
配置过拟合外部获奖文章作独立验证集；保留人工审核环节

六、实施路线图

Phase 1（MVP，2-4 周）
  ├── 单 Agent 审查 (Language + Structure)
  ├── 手动维护 Rubric（静态模板）
  └── 目标：跑通核心审查流程

Phase 2（增强，1-2 个月）
  ├── 完整 7-Agent 体系
  ├── Rubric 模板库（静态版本）
  ├── Calibration Engine v0.5（手工触发）
  └── 目标：覆盖 5+ 竞赛类型

Phase 3（产品化，2-3 个月）
  ├── 导师管理面板 + 批量处理
  ├── Calibration Engine v1.0（全自动 + 人工审核）
  └── 目标：辅导机构内部量产使用

Phase 4（持续优化，长期）
  ├── 校准指标可视化
  ├── 外部获奖文章自动爬取
  ├── 跨竞赛类型的校准知识迁移
  └── 目标：校准闭环完全自动化
