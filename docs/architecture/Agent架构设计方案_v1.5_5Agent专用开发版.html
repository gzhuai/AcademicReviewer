对 7 Agent 的合并说明
A4 = 原 A4 + A7Language & Style（语言修正 + 风格审查）
A5 = 原 A5 + A6Academic Integrity（引用检查 + 原创性检测）

Calibration Engine 位于架构底层，通过分析获奖/失败文章的统计差异（Cohen's d），自动生成配置文件更新建议。教师可将多年辅导经验以文档形式注入校准管道。

二、五个 Agent 详细规格

Agent 1: Rubric Parser（评分标准解析器）

配置层级L1
输入竞赛官方 rubric（PDF/图片/文本）
输出结构化评分维度树 + 权重 + 关键词
配置文件rubric_calibrated.json
校准需求✅ 深 — 维度权重反推、子维度发现
调度轮次Round 1（与 A2/A3 并行）

Agent 2: Structure & Logic（结构与逻辑审查）

配置层级L1
输入全文 + 结构 schema
输出结构完整性报告、段落衔接评估、thesis 一致性评分
配置文件structure_schema_calibrated.json
调度轮次Round 1（与 A1/A3 并行）

Agent 3: Argument & Evidence（论点与证据审查）

配置层级L2 — 赛事类型专属
输入全文 + rubric 维度
输出论证力评估、逻辑谬误清单、fatal_defects 标记
配置文件evidence_patterns.json（按赛事类型）
校准需求✅✅ 最深 — 论证模式反推、每次有新样本即触发

Agent 4: Language & Style（语言与风格审查）— 合并 Agent

合并来源原 A4 (Writing Style) + 原 A7 (Language Polishing)
配置层级L0（语法/拼写） + L1（风格开关）
输出语言修正建议 + 风格批注 + 综合诊断报告
调度轮次Round 2（与 A5 并行）

铁律 — 改写权限边界

操作类型权限示例
拼写/语法/标点✅ 可改写recieve → receive
中式英语（仅语法层面）✅ 可改写不改变作者原意
句式选择/语态分布/用词偏好❌ 仅批注"此句可考虑拆分"
段落节奏/风格判断❌ 仅批注"被动语态占比 65%，建议 40-50%"

Agent 5: Academic Integrity（学术诚信审查）— 合并 Agent

合并来源原 A5 (Citation) + 原 A6 (Originality Guardian)
配置层级L0 + L1
输入全文 + 参考文献列表 + 内部历史稿件库（向量比对）
输出引用完整性报告 + 原创性评分 + 疑似问题标记
调度轮次Round 2（与 A4 并行）

内部子任务 — 两条独立执行路径

子任务 5a：Citation Check（规则引擎优先）
  ├── cite ↔ ref 配对检查
  ├── 引用格式一致性
  └── 输出：结构化配对表

子任务 5b：Originality Check（语义比对）
  ├── 与历史稿件库相似度比对
  ├── AI 生成痕迹检测
  └── 输出：相似度评分 + 可疑段落标记

三、Orchestrator 调度（2 轮制）

══════════════════════════════════════════════════

  A1 Rubric Parser      ← rubric_calibrated.json

八、校准引擎与教师经验注入

8.1 校准引擎流程

Feature Extractor (21维特征) → Cohen's d 效应量 → 统计差异 + 教师经验注入 → Diff Generator → 校准报告 → 配置文件更新

8.2 三模式自适应解析

模式格式要求解析方式LLM 成本
Mode 3 完全结构化### [pattern] feature_name正则精确解析零 LLM
Mode 2 半结构化章节标题分节（如 ## 获奖经验）章节路由 + LLM 逐节抽取1 次 LLM
Mode 1 纯自由格式无格式要求，纯随笔LLM 全文语义解析1 次 LLM

8.3 安全设计

LLM 只负责文本拆分，不负责特征映射——特征映射始终走本地关键词匹配，零幻觉风险。

8.4 Mode 2 章节标题体系

类别标注类型支持的标题（任选其一）
获奖经验pattern获奖经验、获奖规律、成功案例总结、高分特征等
常见问题pitfall常见问题、常见错误、失分原因、扣分点等
高分信号signal高分信号、加分项、评委偏好等
教学方法strategy教学方法、辅导策略、训练方法等
评分标准解读rubric评分标准解读、得分要点分析等

8.5 与 5 Agent 管道的集成

教师经验文档 → 三模式解析 → 校准引擎 → 校准报告 → 配置文件 → 下一轮评审自动使用

九、实施顺序建议

Phase 1 (MVP)        A1 Rubric Parser + A4 Language & Style
Phase 2 (扩展)       A2 + A3 + A5 完整 5 Agent
Phase 3 (校准闭环)    Calibration Engine + 教师经验注入 + 配置自动更新
Phase 4 (产品化)     管理面板 + 批量处理 + 多赛事扩展

十、开发注意事项

  1. A4 的铁律是学生主权的最后防线：prompt 必须写死改写权限边界
  2. A5 尽量多走规则引擎少走 LLM：引用配对用正则比 LLM 更快更准
  3. A1 的 rubric 解析只做一次：结果缓存，不重复解析
  4. A3 和 A4 是 L2 扩展载体：新增赛事类型时主要开发 A3 审查逻辑 + A4 风格模板
  5. 配置文件是唯一数据契约：审查管线不关心配置来源，校准引擎不关心配置消费者
