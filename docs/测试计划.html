一、测试前准备（所有人必做）

1.1 环境检查清单

#检查项命令通过标准
1Python 版本python --version3.10 或更高
2依赖安装pip install -r requirements.txt无报错
3.env 配置确认文件存在且 API Key 非占位值至少 1 个 Key 有效
4数据目录启动后自动创建 data/系统自动处理

1.2 启动服务

在两个终端分别执行：

终端 1 — FastAPI 后端：

python run.py

验证：浏览器打开 http://127.0.0.1:8000/docs，应看到 Swagger UI 文档页面。

终端 2 — Gradio 前端：

python app/gradio_app.py

验证：浏览器打开 http://127.0.0.1:7860，应看到标签页界面。

1.3 测试文件准备

编号用途建议
文件AISEF 科研论文 (.txt)使用已提供的 tests/fixtures/sample_isef_paper.txt
文件BJohn Locke 议论文 (.txt)自己写一段 800 字的观点论述
文件C有引用问题的论文 (.txt)正文引用 [1][2] 但参考文献缺一条
文件D同一论文 (.pdf 或 .docx)测试格式兼容性

三、测试组 2：核心评审流水线（30分钟）

目的：验证 5-Agent 审查流程完整可用。

测试 2.1 标准 ISEF 论文评审（⭐ 最重要）

方式
curl -X POST http://127.0.0.1:8000/api/v1/review \
  -F "file=@tests/fixtures/sample_isef_paper.txt" \
  -F "competition=ISEF" \
  -F "student_name=测试学生"
预期返回 JSON，包含 submission_id、total_score、5 个 Agent 反馈、meta 元数据
通过标准HTTP 200；total_score 在 0-10；5 个 scores 都有数值；所有 Agent 反馈非空；meta 含 word_count/model/duration_seconds

关键验证点：

  • A1 Rubric Parser 正确解析评分标准并给出 rubric_score
  • A2 Structure & Logic 提到论文结构（Abstract, Introduction, Methods, Results, Discussion）
  • A3 Argument & Evidence 提到 p 值、对照组、样本量
  • A4 Language & Style 有两种反馈：rewrites（直接改写）和 suggestions（建议）
  • A5 Academic Integrity 有引用检查结果（正文引用与参考文献交叉匹配）

测试 2.2 Gradio 图形界面提交评审

方式Gradio 「提交评审」：1) 上传文件A; 2) 选择 "ISEF"; 3) 填学生姓名; 4) 选模型; 5) 点"开始评审"
预期右侧出现评分明细和元数据
通过标准显示结果与 API 返回一致；完整 JSON 报告可查看

四、测试组 3：多竞赛类型覆盖（20分钟）

目的：验证每种竞赛类型正确加载专属配置。每个竞赛用 50-100 字短文即可，主要验证配置加载。
编号竞赛类型特殊验证点
3.1John Locke议论文A2 按议论文结构评审；A5 用 MLA 格式
3.2HiMCM数学建模A2 按数学建模结构；A3 关注模型合理性
3.3SIC金融分析A2 按金融分析结构；风格为商业金融体
3.4FBLA商业案例A2 按商业案例结构；风格为商业提案
3.5CTB社会科学A2 按社会科学结构；A5 用 APA 格式
3.6NHD历史研究A5 用 Chicago 格式检查引用

执行方式：在 Gradio 界面或 API 中，每个竞赛提交一次。通过标准：每次均返回完整评审结果，无报错。

五、测试组 4：文件格式兼容性（10分钟）

编号格式操作预期
4.1.txt用文件A正常解析

八、测试组 7：校准引擎（35分钟）

方式
python cli/calibrate.py \
  --competition ISEF --type research \
  --winners data/calibration/winners \
  --losers data/calibration/losers \
  --output data/calibration/report.md
预期控制台输出进展日志；生成 report.md
通过标准report.md 存在；含效应量排序表、配置变更建议、人工复核清单

测试 7.2 交叉验证

准备：在 data/calibration/external/ 下拉入外部获奖文章。

方式加 --external data/calibration/external

通过标准交叉验证表显示各特征的验证状态

通过标准日志显示 detected mode=2；识别 4 章节；每章 ≥1 条 ExpertAnnotation

7.5c Mode 1 纯自由格式

方式使用 data/expert_insights/ISEF_Mode1_freeform_example.md
预期全文 LLM 语义解析

用例操作预期

7.5e 模式检测准确性

用例文档特征预期
