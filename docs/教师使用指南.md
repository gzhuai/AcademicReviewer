# AcademicReviewer 教师使用指南

> **版本**: v0.6.0 | **更新**: 2026-06-13
> **面向用户**: 竞赛指导老师（非技术背景）
> **配套文档**: [使用指南.html](使用指南.html)（系统安装与操作）、[AI_HANDOFF.md](../AI_HANDOFF.md)（开发者参考）

---

## 一、系统简介：它能做什么、不能做什么

AcademicReviewer 是一个 AI 辅助论文评审系统，覆盖 **11 个主流学术竞赛**（ISEF、John Locke、HiMCM、CTB、NHD、FBLA、SIC、WGHS、STS、丘成桐、Marshall Society），用 5 个专业 AI 审查员并行评审学生论文。

### 它能做的（可替代老师）

| 层次 | 内容 | AI 替代度 |
|------|------|----------|
| **形式检查** | 语法、拼写、格式、引用规范 | 95% |
| **结构诊断** | 章节完整性、比例合理性、逻辑衔接 | 85% |
| **论证评估** | 证据匹配、逻辑谬误、反方回应 | 70-80% |

### 它不能做的（需要老师把关）

- 判断实验设计的原创性和科学性（ISEF/丘成桐）
- 判断哲学论证的深度和原创性（John Locke/Marshall）
- 判断估值模型的合理性（SIC/WGHS）
- 根据学生个人水平调整教学策略

> **核心策略**: 让 AI 做它能做的，老师做 AI 做不到的，并通过本系统的教师审核功能最大化协作效率。

## 二、第一次使用：只需 3 步

### Step 1: 安装与启动

```bash
# Windows: 双击运行
scripts\install.bat     # 一键安装（仅首次）
scripts\start_all.bat   # 启动系统

# macOS/Linux:
bash scripts/install.sh
bash scripts/start_all.sh
```

启动后浏览器访问 **http://127.0.0.1:7860**

### Step 2: 配置 API Key

编辑项目根目录的 `.env` 文件，填入至少一个 LLM 提供商的 Key：

```env
DEEPSEEK_API_KEY=sk-你的key
# 或
OPENAI_API_KEY=sk-你的key
# 其他选项: GEMINI_API_KEY, GLM_API_KEY

LLM_PROVIDER=deepseek   # 推荐：性价比最高
```

### Step 3: 提交第一篇论文评审

1. 打开 Gradio 界面 → **「提交评审」** 标签
2. 上传学生论文（支持 .txt / .docx / .pdf）
3. 选择竞赛名称（如 "John Locke"）
4. 点击「开始评审」→ 等待约 30-60 秒
5. 查看评分明细和标注报告
6. 点击「📥 导出批注文档 (.docx)」→ 下载带 Word 批注的原文文档

## 三、核心功能详解

### 3.1 5 个 AI 审查员各负责什么

| 审查员 | 检查内容 | 权重说明 |
|--------|---------|---------|
| **A1 评分标准解析** | 解析竞赛评分细则，建立评分维度树 | 基础参考 |
| **A2 结构逻辑** | 章节完整性、比例合理性、逻辑衔接、内容重复 | John Locke 权重 30%（最重要） |
| **A3 论点证据** | Claim-Evidence 匹配、逻辑谬误、推理链完整性 | ISEF 权重 30%（最重要） |
| **A4 语言风格** | 仅修正拼写/语法/标点错误，不做风格改写 | 各竞赛 10-20% |
| **A5 学术诚信** | 引用匹配检测 + 原创性审查 + AI 痕迹检测 | 各竞赛 15-20% |

> 权重根据竞赛类型自动调整。例如 John Locke 偏结构（A2 占 30%），ISEF 偏证据（A3 占 30%）。

### 3.2 标注报告解读

提交评审后，系统生成带标注的 Markdown 报告。报告结构：

```
# 审稿标注报告

### 段落 1
原文内容...
> **[A4-spelling]** `recieve` → `receive`

### 段落 3
原文内容...
> ⚡**[A2!!—high]** 缺少 counterargument 段落
> **[A3!!—逻辑谬误]** straw_man: ...

---

## 总结
### 结构 (A2) — 7.5/10
...
```

**标注格式说明**:
- `[A4-spelling]` — A4 发现的拼写错误（无前缀 = AI 可完全替代，直接采用）
- `⚡[A2!—medium]` — ⚡ = 建议老师确认后再采用
- `🔴[A5—引用]` — 🔴 = 必须老师亲自审核

| 标记 | 含义 | 老师处理 |
|------|------|---------|
| 无标记 | AI 判断可靠（如拼写、格式） | 直接信任，无需查看 |
| ⚡ | AI 判断需确认（如论证强度） | 快速查看（10-15 秒/条） |
| 🔴 | AI 无法判断（如原创性） | 必须仔细审核（1-2 分钟/条） |

## 四、⭐ 重点：如何填写专家经验（知识卡）

这是提升 AI 评审质量**最关键的一步**。你的一次性 20 分钟投入，可以让后续所有评审的准确性大幅提升。

### 4.1 什么是知识卡

知识卡是你作为竞赛指导老师的经验结晶，系统在每次评审时将其注入 AI 审查员的 prompt，作为判断的参照标准。

知识卡分三层：

| 层级 | 内容 | 必填 | 更新频率 |
|------|------|------|---------|
| **Layer 1: 核心必填** | 优秀标准 + 致命缺陷 + 评分锚点 | 是 | 竞赛标准变化时（一年一两次） |
| **Layer 2: 标杆案例** | 好/差段落对照文本 | 建议 ≥ 1 对 | 每届比赛后补充 2-3 个 |
| **Layer 3: 自由笔记** | 教学策略、随手记 | 否 | 随时 |

### 4.2 文件位置与命名

在 `data/expert_insights/` 目录下创建文件。**支持格式：Markdown（.md）、Word（.docx）、PDF（.pdf）**。建议命名格式：`[竞赛名]_[老师名].md`

> **推荐工作流**: 用 Word 或飞书编辑经验文档 → 导出为 .docx → 拖入 `data/expert_insights/` 目录。系统自动解析其中的文本内容。

```
data/expert_insights/
├── ISEF_张老师.docx          ← 张老师用 Word 写的 ISEF 经验
├── John_Locke_李老师.docx    ← 李老师用飞书导出的 John Locke 经验
├── CTB_social_science.md     ← CTB 经验（Markdown）
├── NHD_history.pdf           ← NHD 经验（PDF）
├── SIC_finance.md            ← SIC/WGHS 经验
└── FBLA_business_case.docx   ← FBLA 经验
```

### 4.3 文档格式（复制即用）

```markdown
## 竞赛: John Locke
## 作者: 李老师
## 经验年限: 5年

### [pattern] 这里是标题
**标题**: 简洁总结你的发现
**描述**: 详细描述你观察到的规律。越具体越好——要有数字、比例、具体案例。
**关联特征**: 如果知道关联的系统特征名，填写。不知道可以留空。

### [pitfall] 这里是标题
**标题**: 学生最常犯的错误
**描述**: 描述什么类型的错误、出现频率、以及正确的做法。
**关联特征**: 可选

### [signal] 这里是标题
**标题**: 高分信号的观察
**描述**: 你注意到的获奖者特有的做法或特征。
**关联特征**: 可选

### [fatal_defect] 
**标题**: 一碰就死的问题
**描述**: 只要出现这个问题的论文，无论其他部分多好，总分必须受限。说明限制多少（如"总分上限 6/10"）。
**关联特征**: 可选

### [scoring_anchor]
**标题**: 9-10分段：描述这个分数段的典型特征
**描述**: 具体描述这个分数段论文的特征。每个分数段写一条。

### [scoring_anchor]
**标题**: 5-6分段：描述这个分数段的典型特征
**描述**: 同上，覆盖至少高分和中等两个分数段。

### [benchmark]
**标题**: 优秀 thesis — 争议性主张 + 论证路径暗示
**描述**: 贴一段真实的优秀写作示例
**为什么好**: 解释为什么这是好的——越具体越好

### [benchmark]
**标题**: 薄弱 thesis — 事实陈述，无法争论
**描述**: 贴一段真实的薄弱写作示例
**为什么差**: 解释问题在哪里

### [strategy]
**标题**: 你常用的教学技巧
**描述**: 记录你的教学方法、训练框架、检查清单等。
**关联特征**: 可选
```

### 4.4 重点：你应该重点填写哪些部分

#### 必填（Layer 1 — 3 类，每类至少 1 条）

**1. 什么是"好"的标准（pattern / pitfall / signal）**

这是最重要的！告诉 AI 你眼中好论文长什么样、差论文长什么样。

> **示例（ISEF 科研型）**:
> - pattern: "获奖者引用密度 15-25 条/千字，初赛淘汰者 < 10 条"
> - pitfall: "中国学生被动语态 > 35% 是主要扣分原因，建议 < 20%"
> - signal: "跨学科引用是 ISEF 决赛信号——几乎所有大奖作品都有"

> **示例（John Locke 思辨型）**:
> - pattern: "逻辑连接词密度 > 8 个/千字是入围关键——获奖者尤其擅长递进和转折类连接词"
> - pitfall: "引言使用 'In today's society...' 是致命开场——评审首 30 秒决定是否继续阅读"
> - signal: "Shortlisted 论文无一例外对反方最强论点做了充分回应，不是简单驳斥而是深入理解后再建设性反驳"

**2. 致命缺陷 / 红线（fatal_defect）**

列出"一碰就死"的问题。这些问题出现时，总分必须受限。

> **示例**:
> - John Locke: "全文无 counterargument → 总分上限 6/10。counterargument 是稻草人 → 论证维度上限 4/10"
> - ISEF: "无 limitations discussion → 方法维度上限 5/10"
> - FBLA: "SWOT 只有列表没有分析 → 战略维度上限 5/10"
> - SIC/WGHS: "估值模型关键假设无理由说明 → 分析维度上限 5/10"

**3. 评分锚点（scoring_anchor）**

至少写 9-10 分段和 5-6 分段的特征描述。这能帮 AI 校准评分尺度。

> **示例（John Locke）**:
> - 9-10分段: "论证链完整且原创性强、counterargument 回应了最强反方、rebuttal 展示了深层推理、结论超出题目本身进行了哲学升华"
> - 5-6分段: "方向大致正确但论证链不完整——有论点但缺乏充分证据或 rebuttal 缺失"

#### 强烈建议（Layer 2 — 至少 1 对标杆案例）

**标杆案例（benchmark）**

给出真实的"好"和"差"段落对比。这是最能直接提升 AI 判断准确性的方式。

> **示例**:
> - 优秀 thesis: "This essay argues that Rawls' difference principle, while normatively appealing, fails to account for motivational constraints — a gap Cohen's critique identifies but does not fully resolve."
>   - **为什么好**: 提出可争议的主张、暗示论证路径、概念使用精准
> - 薄弱 thesis: "Rawls' theory of justice is important. In this essay I will discuss his main ideas."
>   - **为什么差**: 无争议性、无分析性承诺、读者不知道要论证什么

#### 可选（Layer 3 — 随时补充）

**教学策略（strategy）**

记录你的教学方法和框架，不直接进入 AI prompt 但供团队参考。

> **示例**:
> - "PEEL 段落结构法：Point → Evidence → Explanation → Link"
> - "ISEF 方法论写作的 5 段式黄金框架"

### 4.5 工作流：从零到持续优化

```
第一天（15-20 分钟）
├── 打开 data/expert_insights/[竞赛名]_[你的名字].md
├── 填写 Layer 1 核心：至少 3 条 pattern/pitfall/signal + 2 条 fatal_defect + 2 条 scoring_anchor
└── 建议：至少写 1 对 benchmark（好/差对照）

日常使用中（随时 3 分钟）
├── 看到 AI 犯了一个新类型的错误 → 打开文件，加一个 pitfall
├── 发现一个新的高分特征 → 加一个 signal
└── 格式不重要！随手写一段文字丢进去即可，系统自动解析

每届比赛后（30 分钟）
├── 打开 Gradio → 「教师审核」Tab → 「知识卡更新建议」
├── 查看系统自动检测的高频被纠正模式
├── 针对性补充或修正知识卡
└── 审核 10-20 篇 AI 评审结果，发现反复判错的地方 → 修正
```

## 五、教师审核工作流（重要！）

### 5.1 为什么需要审核 AI 的评审结果

AI 的判断不是 100% 准确。通过你的一次次审核，系统会：
- 统计哪些类型的判断 AI 做得好（可以自动化）
- 发现哪些类型的判断 AI 反复出错（需要补充知识卡）
- 积累数据用于未来自动校准 AI 的置信度

### 5.2 审核流程（每次 2-5 分钟）

1. **打开 Gradio** → **「教师审核」** 标签
2. **输入 Submission ID**（从评审历史中获取）→ 点击「加载评审」
3. **查看标注报告**：展开「📄 标注报告」查看完整的带批注原文
4. **查看审核项明细**：系统自动列出所有 AI 审查意见，按置信度分类
   - FULL（无标记）= 可直接信任，无需操作
   - REVIEW（⚡）= 快速确认，约 10-15 秒/条
   - ESCALATE（🔴）= 必须仔细审核，约 1-2 分钟/条
5. **两种操作方式**：
   - **快速通道**: 如果 AI 判断整体靠谱 → 点击「全部确认」
   - **精确覆写**: 如果某些条目需要修正 → 在「覆写说明」中输入 JSON
6. **提交反馈**

### 5.3 覆写说明格式

```json
[
  {"index": 0, "action": "OVERRIDE", "note": "rebuttal 确实回应了核心主张但推理步骤不完整"},
  {"index": 3, "action": "REFINE", "note": "这里不是 straw_man，是 weak_analogy"},
  {"index": 7, "action": "CONFIRM", "note": ""}
]
```

- `index`: 审核项序号（从审核项表格第一列获取）
- `action`: CONFIRM（确认）/ OVERRIDE（修正：AI 判断错误）/ REFINE（细化：AI 方向对但不完整）
- `note`: 你的修正说明，越具体越好

### 5.4 查看反馈统计

在「教师审核」标签底部，点击「加载统计」查看：
- 整体确认率（你认可 AI 判断的比例）
- 按 Agent 的确认率（哪个 Agent 最靠谱/最不靠谱）
- 按置信度标签的确认率（AI 标 FULL 的你真的都确认了吗？）

### 5.5 知识卡更新建议

在「教师审核」标签底部，点击「检测建议」：
- 系统自动发现哪些 Agent + 条目类型被反复纠正
- 给出建议的知识卡类型（fatal_defect / pattern / pitfall / benchmark）
- 附带你的历史备注作为参考

## 六、团队协作模式（可选）

如果你们是多人团队，可以让所有老师本地运行系统，数据自动汇总到一个中央实例。

1. **管理员**部署一台中央服务器（设置 `HOST=0.0.0.0`）
2. **同事**在 `.env` 中配置：
   ```env
   SYNC_SERVER_URL=http://管理员IP:8000
   INSTANCE_NAME=你的名字
   ```
3. 同事每次提交评审或运行校准，结果自动上报
4. 管理员在「管理看板」查看所有汇总数据

## 七、常见问题

**Q: 我写的专家经验文件多久生效？**
A: 下次提交评审时自动生效，无需重启。

**Q: 我可以同时有多个经验文件吗？**
A: 可以。系统会合并同一竞赛的所有 `.md` 文件。建议每人一个文件方便管理。

**Q: 知识卡会影响哪些审查员？**
A: 当前注入 A2（结构）和 A3（论证）。这两个 Agent 最能从领域经验中受益。

**Q: 学生论文数据安全吗？**
A: 论文全文发送到 LLM API（DeepSeek/OpenAI/Gemini/GLM）进行处理。请勿上传包含个人敏感信息的学生作品。评审结果仅在本地 SQLite 数据库存储。

**Q: 反馈闭环有什么用？**
A: 每一次审核都不浪费——你的确认/修正在统计层面校准 AI 的置信度，在内容层面优化知识卡。长期来看，AI 的准确率会随你的使用而提升。

| 竞赛 | 类型 | 关键权重 | 知识卡重点 |
|------|------|---------|-----------|
| ISEF / STS | 科研型 | A3 证据 30% | 实验设计规范、统计方法、局限性讨论 |
| 丘成桐 | 进阶科研 | A3 证据 30% | 同上，标准更高 |
| John Locke | 思辨型 | A2 结构 30% | 反方回应质量、论证链完整性、哲学深度 |
| Marshall | 思辨型(经济) | A2 结构 30% | 同上 + 经济模型应用 |
| HiMCM | 数模型 | A3 证据 30% | 模型假设合理性、灵敏度分析、公式正确性 |
| CTB | 社科型 | A2/A3 均衡 | 调研方法论、跨学科视角、方案可行性 |
| NHD | 历史型 | A3 证据 30% | 一手史料、史学解读、时代语境还原 |
| SIC / WGHS | 金融型 | 均衡分布 | 估值假设合理性、数据可溯性、风险量化 |
| FBLA | 商科型 | A2 结构 30% | 框架应用正确性、财务预测合理性、竞品分析 |
